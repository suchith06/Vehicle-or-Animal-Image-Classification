Vehicle & Animal Classifier

This is a web application that analyzes uploaded images and classifies them into three categories: Vehicles (Cars, Trucks, Planes, Boats), Animals (Dogs, Cats, Birds, Elephants), or Unknown (objects that do not belong to either category). Users can simply drag and drop images or upload JPG/PNG files (up to 5 MB). The application works entirely in the browser and does not require a backend server.

What It Does

Upload an image (JPG or PNG), and the app analyzes it to classify the main object as:

🚗 Vehicle: Cars, trucks, motorcycles, planes, boats, etc.
🐾 Animal: Dogs, cats, birds, elephants, etc.
❓ Unknown: Objects that do not fit into the above categories

The app displays a confidence score along with the top 5 predicted labels generated by the model.

How to Run Locally
Option 1: Open Directly

Open index.html directly in your browser.

Option 2: Using Python Server

After navigating to the project directory:

How the Classification Works

Model: MobileNet v2 using TensorFlow.js

Steps:
Upload an Image
Supported file types: JPG/PNG
Maximum file size: 5 MB
Model Inference
MobileNet v2 analyzes the image and generates predictions from 1,000 ImageNet classes.
Post-processing
The application maps the predicted classes into Vehicle, Animal, or Unknown categories using keyword matching.
Results
The application displays:
Category
Confidence score (%)
Top 5 predicted labels
How to Test It
Test Case 1: Vehicle Classification

Upload: Car, truck, motorcycle, or airplane images

Expected Result: Vehicle classification (70–95% confidence)

Check: Top 5 predictions should contain vehicle-related labels.

Test Case 2: Animal Classification

Upload: Dog, cat, bird, horse, or elephant images

Expected Result: Animal classification (70–95% confidence)

Check: Breed or animal type should appear in the predictions.

Test Case 3: Unknown Classification

Upload: Furniture, food, people, or building images

Expected Result: Unknown classification

Check: The application should handle the input gracefully without crashes.

Known Limitations
Model Limitations
ImageNet Bias
Only recognizes the 1,000 classes available in the ImageNet training dataset.
Single Object Detection
Only classifies the dominant object in an image, not multiple objects.
Small or Distant Objects
Tiny or far-away objects may not be recognized accurately.
Novel Objects
Rare or unseen objects may not be classified correctly.
Confidence Scores Are Not Absolute
An 85% confidence score does not necessarily mean an 85% probability of correctness.
Application Limitations
No GPU Support
Runs on CPU only and may take 1–5 seconds on older devices.
Maximum File Size: 5 MB
Prevents uploading excessively large files.
Supported Formats Only: JPG/PNG
GIF, WebP, and BMP are not supported.
Internet Required for Initial Load
Internet access is only required the first time to download the model; afterward, it is cached.
No Image Storage
Uploaded images are not saved.
Not Customizable
The model cannot be retrained within the application.
Resolution Limitations
Very high-resolution images (8000+ pixels) may slow performance.
Edge Cases
Toy vehicles → Often classified as Unknown
Horse carriages → May be classified as either Vehicle or Animal due to ambiguity
Drawings/Cartoons → Often classified as Unknown
Stuffed animals → Usually classified as Unknown because they are detected as toys rather than animals

How Classification Works
Model Architecture

The application uses MobileNet v2, a lightweight pre-trained convolutional neural network trained on the ImageNet dataset (1,000+ object classes).

Why MobileNet v2?
Lightweight (~100 MB) and optimized for browser execution
Fast inference (~1–2 seconds per image)
Good accuracy for general object recognition tasks
Runs entirely in the browser without requiring a backend server
